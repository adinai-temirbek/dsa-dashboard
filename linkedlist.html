🔗 Linked List Data Structure

A linked list is a dynamic data structure where each node holds data and a pointer to the next node. Unlike arrays, nodes are not stored in contiguous memory.

📖 Overview

Linked lists consist of nodes, each containing a data value and a pointer (reference) to the next node. The first node is the head; the last node points to null. They allow efficient insertion and deletion at any position — no shifting required!

💡 Node Structure
Each node = [DATA | NEXT →]. The head pointer points to the first node. The last node's NEXT pointer is NULL, marking the end of the list.

📝 Pseudocode — Core Operations

class Node: data = value next = null class LinkedList: head = null function insertAtHead(value): node = new Node(value) node.next = head head = node // O(1) function insertAtTail(value): node = new Node(value) if head == null: head = node; return curr = head while curr.next != null: curr = curr.next curr.next = node // O(n) function delete(value): if head.data == value: head = head.next; return curr = head while curr.next != null: if curr.next.data == value: curr.next = curr.next.next // Skip the node return curr = curr.next

🌐 Real-World Applications

  • Music/Video Playlists: Each song links to the next; doubly-linked enables prev/next navigation
  • Browser Navigation: Forward/back history implemented as a doubly linked list
  • Memory Allocators: Free lists in OS memory management
  • Polynomial Arithmetic: Each term is a node; used in symbolic math systems
  • Undo Buffers: Each action stored as a node in an editor's history list
The Linked List was invented by Allen Newell, Cliff Shaw, and Herbert Simon around 1955–1956 as part of their IPL (Information Processing Language) for AI research. The concept was radical at the time — using memory addresses as pointers was a new idea. Their work eventually contributed to Simon and Newell winning the Turing Award in 1975.
