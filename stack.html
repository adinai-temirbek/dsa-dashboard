📚 Stack Data Structure

A stack is a linear data structure following the LIFO (Last In, First Out) principle — like a stack of plates. Push adds to the top; Pop removes from the top.

📖 Overview

A stack is one of the most fundamental data structures in computer science. It supports three primary operations: push (insert element on top), pop (remove element from top), and peek (view top without removing). All three run in O(1) time.

💡 LIFO Principle
The last element pushed onto the stack is the first one to be popped off. Think of a stack of books — you can only take from the top, and you add to the top.

🔄 LIFO Demonstration
Push: A → B → C → D. Then Pop: D is removed first (last in, first out). Order of pops: D, C, B, A.

🧮 Expression Evaluation with Stack

Stacks are used to evaluate arithmetic expressions. Here's how the expression 3 + 4 * 2 is processed:

STEP 1
Read 3
Push to operand stack
STEP 2
Read +
Push to operator stack
STEP 3
Read 4
Push to operand stack
STEP 4
Read * (higher prec.)
Push to operator stack
STEP 5
Read 2, end
Evaluate: 4*2=8, then 3+8=11

📝 Pseudocode

// Stack implementation using array class Stack: stack = [] top = -1 function push(element): stack[++top] = element // Add to top function pop(): if isEmpty(): throw "Underflow" return stack[top--] // Remove from top function peek(): if isEmpty(): throw "Empty" return stack[top] // View top without removing function isEmpty(): return top == -1

⚖️ Advantages & Disadvantages

  • All primary operations are O(1)
  • Simple and easy to implement
  • Memory efficient when using arrays
  • Ideal for tracking state (undo/redo)
  • No random access — only top is accessible
  • Fixed size limitation with array implementation
  • Stack overflow if not bounded

🌐 Real-World Applications

  • Function Call Stack: Your CPU uses a call stack to manage function calls and returns
  • Browser History: The Back button uses a stack (current page = top)
  • Undo/Redo: Text editors and Photoshop use stacks for operation history
  • Expression Evaluation: Compilers use stacks to parse and evaluate expressions
  • Parenthesis Matching: Checking balanced brackets in code validation
When you call a function in any programming language, the computer automatically pushes a "stack frame" containing the function's local variables, parameters, and return address onto the call stack. When the function returns, that frame is popped. A "Stack Overflow" error occurs when too many nested function calls exhaust the stack's memory — which is where the famous Q&A website got its name!
