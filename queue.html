🎯 Queue Data Structure

A queue is a linear data structure following the FIFO (First In, First Out) principle — like a real-world line. Elements enter from the rear and leave from the front.

📖 Overview

Queues support two primary operations: Enqueue (add to rear) and Dequeue (remove from front). Just like people queuing at a ticket counter — the first person to arrive is the first to be served.

💡 FIFO Principle
FRONT ← [A][B][C][D] ← REAR
Enqueue E: FRONT ← [A][B][C][D][E] ← REAR
Dequeue: Removes A from FRONT → [B][C][D][E]

🔄 Circular Queue

A circular queue connects the last position back to the first, solving the wasted space problem of a standard queue. When the rear reaches the end, it wraps around using modulo arithmetic.

📝 Pseudocode

class Queue: queue = [] front = 0 rear = -1 function enqueue(element): rear = rear + 1 queue[rear] = element // Add to rear — O(1) function dequeue(): if isEmpty(): throw "Underflow" element = queue[front] front = front + 1 return element // Remove from front — O(1) function peek(): return queue[front] // View front without removing // Circular Queue: rear = (rear + 1) % size

🗂️ Types of Queues

Simple Queue

Basic FIFO. Elements added to rear, removed from front. Can waste space after dequeues.

Circular Queue

Rear wraps around to front using modulo. Efficient memory utilization. Used in buffering.

Priority Queue

Elements dequeue in priority order, not arrival order. Implemented with a Heap. Used in Dijkstra's algorithm.

Deque (Double-Ended)

Enqueue and Dequeue from both ends. Combines Stack + Queue. Used in sliding window algorithms.

🌐 Real-World Applications

  • CPU Scheduling: Ready queue for process scheduling in operating systems
  • BFS Graph Traversal: Breadth-first search uses a queue to explore level by level
  • Print Spooler: Printer queue processes jobs in the order received
  • Network Packets: Routers use queues to buffer packets during congestion
  • Keyboard Buffer: Keystrokes are stored in a queue to preserve input order
The Breadth-First Search (BFS) algorithm — used to find shortest paths in graphs, social network connections, and GPS navigation — fundamentally relies on a queue. By processing nodes level by level using FIFO order, BFS guarantees the shortest path is found first in unweighted graphs.
