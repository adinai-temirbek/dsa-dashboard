⚙️ Sorting Algorithms

Sorting algorithms arrange elements in a specific order. Visualize Bubble Sort, Selection Sort, Insertion Sort, and Merge Sort with animated bar charts.

📖 Overview

Sorting is fundamental to computer science — sorted data enables binary search, improves cache performance, and is required by many algorithms. Each sorting algorithm makes different trade-offs between time complexity, space complexity, and stability.

💡 Key Concept
A stable sort preserves the relative order of elements with equal keys. Merge Sort is stable; Selection Sort is not. Stability matters when sorting objects by multiple fields.

📝 Bubble Sort

Repeatedly compare adjacent elements and swap them if they're in the wrong order. After each pass, the largest unsorted element "bubbles up" to its correct position.

function bubbleSort(arr): n = arr.length for i = 0 to n-1: for j = 0 to n-i-2: if arr[j] > arr[j+1]: swap(arr[j], arr[j+1])

📝 Selection Sort

Find the minimum element in the unsorted portion and place it at the beginning. Repeat until fully sorted. Makes O(n) swaps — better than Bubble Sort when writes are expensive.

function selectionSort(arr): n = arr.length for i = 0 to n-1: minIdx = i for j = i+1 to n-1: if arr[j] < arr[minIdx]: minIdx = j swap(arr[i], arr[minIdx])

📝 Insertion Sort

Build the sorted array one element at a time by inserting each new element into its correct position. Excellent for small or nearly-sorted arrays — used in practice for small subarrays within larger sorts.

function insertionSort(arr): for i = 1 to n-1: key = arr[i] j = i - 1 while j >= 0 and arr[j] > key: arr[j+1] = arr[j] j = j - 1 arr[j+1] = key

📝 Merge Sort

Divide the array in half recursively until single elements remain, then merge sorted halves back together. Guarantees O(n log n) in all cases. Stable and widely used in production systems.

function mergeSort(arr): if arr.length <= 1: return arr mid = floor(arr.length / 2) left = mergeSort(arr[0..mid]) right = mergeSort(arr[mid..end]) return merge(left, right) function merge(left, right): result = [] while both arrays have elements: if left[0] <= right[0]: append left[0] else: append right[0] append remaining elements return result

📝 Radix Sort

function radixSort(arr): if arr.length ≤ 1: return max = findMax(arr) exp = 1 while floor(max / exp) > 0: count = new array[10] initialized to 0 output = new array[arr.length] // Count occurrences of current digit for i = 0 to arr.length-1: digit = floor(arr[i] / exp) % 10 count[digit]++ // Build cumulative count for i = 1 to 9: count[i] += count[i-1] // Build output array (stable sort) for i = arr.length-1 down to 0: digit = floor(arr[i] / exp) % 10 output[count[digit]-1] = arr[i] count[digit]-- arr = output exp *= 10

📝 Quick Sort

function quickSort(arr, low, high): if low ≥ high: return pivotIndex = partition(arr, low, high) quickSort(arr, low, pivotIndex - 1) quickSort(arr, pivotIndex + 1, high) function partition(arr, low, high): pivot = arr[high] // choose last element as pivot i = low - 1 for j = low to high-1: if arr[j] ≤ pivot: i++ swap(arr[i], arr[j]) swap(arr[i+1], arr[high]) // put pivot on the right place return i+1

⚖️ Advantages & Disadvantages

  • Bubble/Insertion: Simple to understand and implement
  • Insertion Sort: Adaptive — very fast on nearly-sorted or small data
  • Merge Sort: Guaranteed O(n log n), stable, predictable performance
  • Quick Sort: Extremely fast in practice (best real-world average case)
  • Quick Sort: In-place (almost no extra memory), excellent cache performance
  • Counting Sort: Linear time O(n + k) — beats comparison sorts for integers
  • Radix Sort: Linear time O(nk) for fixed-length keys (e.g. integers, strings)
  • Radix Sort: Non-comparison based — can be faster than Quick/Merge for certain data
  • Bubble/Selection/Insertion: O(n²) — very slow on large unsorted data
  • Merge Sort: Requires O(n) extra space (not in-place)
  • Quick Sort: Worst-case O(n²) if pivot chosen poorly (e.g. sorted data)
  • Quick Sort: Not stable — relative order of equal elements can change
  • Counting Sort: Limited to integers or small range of values (needs extra space O(k))
  • Radix Sort: Requires stable sort per digit (extra space or time), works best on fixed-length keys
  • Non-comparison sorts (Counting/Radix): Cannot be used on arbitrary objects

🌐 Real-World Applications

  • Merge Sort: Used in Python's Timsort (hybrid algorithm), Java's Arrays.sort for objects
  • Insertion Sort: Used internally for small arrays by many sorting libraries
  • Sorting in general: Database query optimization, file system organization, e-commerce product sorting
  • Sorting in general: Genomics (sorting DNA sequences), rendering pipelines (depth sorting)
No comparison-based sorting algorithm can beat O(n log n) in the worst case — this is mathematically proven! However, non-comparison sorts like Radix Sort and Counting Sort can sort integers in O(n) time by exploiting the structure of the data rather than comparing elements.
