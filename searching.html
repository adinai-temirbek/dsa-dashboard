🔍 Searching Algorithms

Searching algorithms locate a specific element within a data structure. The two fundamental approaches are Linear Search (sequential) and Binary Search (divide and conquer).

📖 Overview

Searching is one of the most common operations in computing. Choosing the right search algorithm depends on whether the data is sorted, the expected size of the dataset, and performance requirements. Linear search works on unsorted data but checks every element. Binary search is dramatically faster but requires sorted data.

💡 Key Insight
Binary Search cuts the search space in half with each comparison. For 1,000,000 elements, it takes at most ~20 comparisons vs up to 1,000,000 for Linear Search.

📝 Pseudocode — Linear Search

function linearSearch(arr, target): for i = 0 to arr.length - 1: if arr[i] == target: return i // target found at index i return -1 // target not found

📝 Pseudocode — Binary Search

function binarySearch(arr, target): lo = 0 hi = arr.length - 1 while lo <= hi: mid = floor((lo + hi) / 2) if arr[mid] == target: return mid // found! else if arr[mid] < target: lo = mid + 1 // search right half else: hi = mid - 1 // search left half return -1 // not found

⚖️ Advantages & Disadvantages

Linear Search

  • Works on unsorted data
  • Simple to implement
  • No preprocessing required
  • Slow for large datasets O(n)
  • Inefficient compared to alternatives

Binary Search

  • Very fast: O(log n)
  • Ideal for large sorted datasets
  • Eliminates half the data each step
  • Requires sorted array
  • Poor for linked lists (no random access)

🌐 Real-World Applications

  • Linear Search: Scanning unsorted records, contact list search on small datasets
  • Binary Search: Dictionary lookups, database index searches, Git bisect for bug finding
  • Binary Search: Finding words in sorted dictionaries, autocomplete systems
  • Binary Search: Square root estimation algorithms, OS memory allocation
Binary Search was first published in 1946, but the first bug-free implementation wasn't written until 1962! Even Jon Bentley (who popularized it) noted that most programmers fail to implement it correctly on the first try. The tricky part is computing the midpoint — integer overflow can occur with (lo + hi) / 2; the safe version is lo + (hi - lo) / 2.
