#️⃣ Hash Functions & Hash Tables

Hash tables provide O(1) average-case lookup, insertion, and deletion by mapping keys through a hash function to array indices.

📖 Overview

A hash table is a data structure that uses a hash function to compute an index into an array of buckets. The hash function transforms a key into a number used as the array index. When two keys map to the same index — a collision — it must be resolved through techniques like separate chaining or open addressing.

💡 Hash Function Formula
For this demo: hash(key) = key % tableSize
Example: key=23, size=7 → 23 % 7 = 2 → stored at index 2

📝 Pseudocode

class HashTable: SIZE = 7 table = array of empty lists (size = SIZE) function hash(key): return key % SIZE // Simple modulo hash function insert(key): index = hash(key) table[index].append(key) // Separate chaining function search(key): index = hash(key) return key in table[index] // Search only in bucket function delete(key): index = hash(key) table[index].remove(key)

🛡️ Collision Resolution Methods

Separate Chaining

Each bucket holds a list. Colliding elements are appended to the list at the same index. Lookup requires scanning the chain.

  • Simple to implement
  • No clustering issues
  • Table never "fills up"
  • Extra memory for linked list nodes
  • Poor cache performance

Open Addressing

If a bucket is taken, probe for the next available slot using Linear Probing (i+1), Quadratic Probing (i²), or Double Hashing.

  • Better cache performance
  • No pointer overhead
  • Clustering can degrade performance
  • Deletion is tricky

🌐 Real-World Applications

  • Dictionaries / Maps: Python dicts, JavaScript objects use hash tables internally
  • Database Indexing: Hash indexes for O(1) record lookup by primary key
  • Caches (DNS, CDN): Fast lookup of cached responses using hashed keys
  • Cryptography: Cryptographic hash functions (SHA-256) for data integrity
  • Compilers: Symbol tables storing variable names and their attributes
Python's dictionary was so well-designed that it was famously called "the fastest hash table implementation." In Python 3.6+, dicts also preserve insertion order — a combination of hash table efficiency AND linked list ordering. The average load factor before resizing is 2/3, balancing memory efficiency with collision frequency.
